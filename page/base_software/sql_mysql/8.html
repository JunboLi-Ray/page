mysql分为服务层和存储层 存储层innodb支持大多数类型，没有特殊要求就innodb 可以用许多开源工具，来进行mysql测试，主要也是执行时间等参数。 数据结构： char定长varchar不定长还带一位存长度 datetime存时间，范围很大。timestamp只有70年到23年。 索引 多列索引，索引先后位置不同，对查询影响不同。 聚簇索引，将数据全都在叶子节点。 覆盖索引，是说索引包含了要查询的数据，减少磁盘过程，直接内存读取。 查询的根据索引排序，则可以用索引展示排序结果，而不用存储的排序功能。 存储会有碎片，可以用optimize table或改变表消除，但要注意再更改时的分页。
